# 제안

## 방안 1: 문서에 상세 지침 추가
- README/AGENT에 Apple 플랫폼, 한국어 주석, 테스트 및 에이전트 활용 지침을 정리한다.
- 장점: 문서 진입점이 하나로 정리되어 유지관리 쉬움.
- 리스크: 잘못 작성하면 중복 또는 오해 소지, 정기 검토 필요.

## 방안 2: 별도 가이드 문서 생성
- docs/ 또는 wiki에 Apple/에이전트 가이드 문서를 별도 생성하고 링크한다.
- 장점: 문서 분리로 집중도 향상, 확장 유연.
- 리스크: 링크 누락, 업데이트 일관성 낮음.

## 선택 이유
- README/AGENT가 핵심 진입문서이므로 해당 내용이 바로 들어가야 하고, 유지할 책임자가 이미 존재.

## 리스크 관리
- README/AGENT 내용 정기 검토를 문서 하단 리비전 히스토리에 기록.
- 중복을 최소화하기 위해 변경 시 동시에 양쪽 문서를 수정하고 검증한다.
