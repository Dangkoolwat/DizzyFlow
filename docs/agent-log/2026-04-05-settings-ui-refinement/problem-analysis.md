# 문제 분석: 설정(Settings) UI/UX 불일치 및 가독성 저하

## 1. 개요
DizzyFlow의 앱 설정 화면은 사용자에게 다양한 옵션을 제공하지만, 초기 목업 단계에서 디자인 일관성이 부족하고 일부 UI 요소가 불필요하게 공간을 차지하는 문제가 발견되었습니다.

## 2. 세부 문제 항목
- **테마 전환기**: 상단 툴바의 테마 피커가 텍스트와 함께 표시되어 시각적으로 번잡하며, 공간을 비효율적으로 사용함.
- **불필요한 카테고리**: 프로젝트 방향성 변경으로 인해 `Gemma4` 설정이 더 이상 필요하지 않으나 메뉴에 남아 있음.
- **일반 설정 레이아웃**: 언어 선택 드롭다운이 지나치게 길어 디자인 밸런스가 맞지 않고, 테마 설정이 툴바와 중복되어 사용자 혼란 가능성이 있음.
- **모델 관리 화면**: `Sherpa-onnx` 및 `WhisperKit` 하위의 여러 모델을 관리하기에 기존 단일 행 레이아웃이 협소함. 특히 긴 모델 이름 대응이 어려움.
- **라이선스 화면**: 라이선스 뷰 선택 시 창의 너비가 비정상적으로 확장되는 레이아웃 버그가 존재함.
- **사용자 경험(UX)**: 사이드바의 Settings 버튼 클릭 시 열기/닫기(Toggle) 동작이 일관되지 않음.

## 3. 제약 사항
- macOS 14/15 환경에서의 네이티브 UI 스타일 유지.
- `WorkflowStore` 전역 상태와의 동기화 보장.
- 외부 `license.txt` 파일 로드 시의 안정성 확보.
