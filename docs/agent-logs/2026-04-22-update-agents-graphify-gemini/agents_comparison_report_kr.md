# AGENTS.md 비교 분석 리포트: Graphify 도입 전후

## 1. 개요
`graphify`의 도입은 `AGENTS.md` 운영 표준을 **"직관 중심의 개발"**에서 **"데이터 중심의 시스템 엔지니어링"**으로 전환하는 중요한 분기점입니다. 구조화된 지식 그래프 분석을 필수 워크플로우에 통합함으로써, 핵심 아키텍처 패턴이 훼손될 위험을 획기적으로 줄였습니다.

## 2. 주요 차이점 비교

| 항목 | 도입 이전 (Standard) | 도입 이후 (Systemic) |
| :--- | :--- | :--- |
| **분석 방식** | 에이전트의 내부 추론과 파일 탐색에 의존 | **God Nodes** 및 의존성 리포트 분석 필수화 |
| **리스크 인지** | 일반적인 "Non-trivial" 작업 목록 기반 | 프로젝트별 **고위험 구역**(`WorkflowStore` 등) 명시 |
| **상태 관리** | 원칙으로서의 SSOT 언급 | `WorkflowPhase` 전이 분석을 통한 실행적 SSOT 보장 |
| **검증 방식** | 변경 사항 나열 중심의 보고 | **빌드 안정성** 및 회귀 테스트 체크리스트 중심 |
| **에이전트 호환성** | 한/영 혼용 (맥락 손실 가능성 있음) | **표준화된 영어** (글로벌 에이전트 최적화) |

## 3. 도입 후 장점 분석

### A. 외과수술급 정밀 수정 (Surgical Precision)
가장 연결성이 높은 컴포넌트인 "God Nodes"를 식별함으로써, 에이전트는 어떤 파일이 "수정 시 극도의 주의가 필요한지" 명확히 알게 됩니다. 이는 `WorkflowStore`와 같은 복잡한 상태 머신에서 발생할 수 있는 연쇄 오류를 방지합니다.

### B. 글로벌 에이전트 협업 최적화
구조화된 고정밀 영문 형식을 채택함으로써, OpenAI, Gemini, DeepSeek 등 어떤 AI 모델이 투입되어도 기본 학습 데이터 언어에 관계없이 지침을 100% 명확하게 이해할 수 있습니다.

### C. 기술 부채 감소
필수 빌드 체크와 사이드 이펙트 분석은 에이전트가 "당장의 작업 해결"을 넘어 "코드베이스의 장기적 건강"을 고려하게 만듭니다.

## 4. 단점 및 트레이드오프

### A. 분석 지연 (Latency)
에이전트가 코드를 작성하기 전 리포트를 읽는 "분석 단계"를 거쳐야 하므로, 구현 시작까지의 시간이 약간 늘어납니다.

### B. 그래프 유지보수
지식 그래프는 스냅샷입니다. 구조적 변경이 발생할 때마다 `updateGraphify`를 실행하여 "진실의 원천"을 최신화해야 하는 관리 비용이 발생합니다.

## 5. 최종 결론
**Post-Graphify AGENTS.md**는 DizzyFlow와 같이 미션 크리티컬한 macOS 애플리케이션에 최적화된 표준입니다. 이는 AI 에이전트를 단순한 "코더"에서 "시니어 아키텍트"로 변모시키며, 분석 시간의 미세한 증가는 아키텍처 무결성 보장이라는 거대한 이득에 비하면 매우 작은 대가입니다.
