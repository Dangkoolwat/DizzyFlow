
# DizzyFlow UX Flow Notes

## Idle and Ready
Last updated: 2026-04-04

이 문서는 DizzyFlow의 초기 프로토타입에서 Idle / Ready 단계에 대해
채팅 논의로 확정된 UX 기준을 다른 에이전트가 오해 없이 이해하도록 정리한 문서다.

---

## 1. 기본 전제

DizzyFlow는 단순 전사 앱이 아니라 workflow-first subtitle platform이다.

초기 프로토타입에서 중요한 것은
기능을 많이 넣는 것보다 사용자가 작업 흐름을 자연스럽게 이해하고,
상태 전이가 안정적으로 보이도록 만드는 것이다.

현재 기준 프로세스는 다음 6단계다.

    Idle
    Ready
    Processing
    Completed
    Failed
    Cancelled

이 문서는 그중 Idle / Ready까지만 정리한다.

---

## 2. Home과 Sidebar의 의미

Home은 프로그램 시작 시 기본 진입 위치다.

앱이 실행되면 사용자는 기본적으로 Home에 들어온다.

사이드바에는 이전에 생성된 문서(Document) 폴더들이 표시된다.
이 문서들은 과거 작업 결과를 담고 있는 항목이며,
각 문서 폴더에는 상태 저장 파일이 존재한다.

초기 버전에서 문서 상태는 최소 다음 중 하나다.

    Completed
    Failed
    Cancelled

사용자가 사이드바에서 기존 문서를 클릭하면,
중앙 Workspace는 해당 문서의 저장 상태에 따라 다른 내용을 보여준다.

    Completed
    - SRT 미리보기
    - 결과 관련 버튼 표시

    Failed
    - 오류 메시지 출력
    - 어디서 오류가 났는지 확인 가능한 디버깅성 정보 표시

    Cancelled
    - 사용자 취소 안내 메시지 표시
    - 다시 시작 버튼 제공
    - 다시 시작 시 기존 문서를 기준으로 Processing을 다시 수행
    - 최종 성공 시 Completed 상태로 갱신 가능

즉 Home은 단순 새 작업 화면만이 아니라,
기존 문서 확인과 새 작업 진입이 함께 존재하는 시작 위치다.

---

## 3. Idle 단계 정의

Idle은 앱의 기본 시작 상태이며,
아직 새 작업이 시작되지 않은 상태다.

이 단계의 핵심은
"파일을 넣기 전에 필요한 주요 설정을 미리 잡을 수 있게 하는 것"이다.

### Idle에서 사용 가능한 주요 설정

Idle 단계의 메인 Workspace 상단에는 다음 4개 설정 UI가 있다.

    FPS
    언어
    FCP 템플릿
    전사 모델

이 설정은 사용자가 파일을 넣기 전에도 미리 조정할 수 있어야 한다.

또한 이 값들은 마지막 사용값을 기억하고,
다음 실행 시 초기값처럼 표시된다.

이 기본값 기억은 편의 기능이며,
사용자는 Idle과 다음 단계(Ready)에서도 계속 수정할 수 있다.

### Idle의 화면 역할

Sidebar
- 기존 문서 목록 표시
- 사용자는 과거 문서를 선택해 상태를 다시 확인할 수 있음

Main Workspace
- 새 작업 시작 전 대기 화면
- 파일 드롭 또는 파일 선택 UI 표시
- 상단에 주요 설정 4개 표시
- 중앙은 파일 입력을 유도하는 영역

Inspector
- 기본 팁 표시
- 현재 설정에 대한 설명 표시
- 필요 시 설정 변경에 따른 안내 문구 표시

### Idle에서 아직 하지 않는 것

Idle에서는 아직 실제 작업이 시작되지 않는다.
또한 이 단계에서는 새 Document 폴더를 생성하지 않는다.

중요:
새 Document 폴더는 파일을 넣는 시점이 아니라
사용자가 시작 버튼을 눌러 실제 작업을 시작하는 시점에 생성된다.

---

## 4. Ready 단계 정의

Ready는 파일이 업로드되었고,
이제 사용자가 시작 버튼을 누르면 다음 단계로 진행할 수 있는 상태다.

즉 Ready는 실행 전 검토 상태다.

핵심은 다음과 같다.

    파일은 이미 선택되었다
    주요 설정은 여전히 수정 가능하다
    아직 실제 프로세스는 돌지 않는다
    시작 버튼을 눌러야 Document 생성 + Processing 진입이 일어난다

### Ready 진입 조건

사용자가 Idle 상태에서 파일을 드롭하거나 파일 선택을 완료하면
화면은 Ready 상태로 전환된다.

하지만 이 시점에서도 아직 Document 폴더는 없다.

### Ready의 화면 역할

Main Workspace 상단
- Idle과 동일하게 주요 설정 4개 유지
- 사용자는 이 상태에서도 계속 수정 가능

Main Workspace 중앙
- 무거운 메타정보를 전부 보여줄 필요는 없음
- 간단한 파일 정보만 표시
- 파일명만 텍스트로 두기보다 시각적 확인감을 주는 것이 좋음
- 가능하면 영상 첫 프레임 썸네일 또는 파일 아이콘을 함께 표시
- "이 설정으로 시작할까요?" 같은 맥락성 문구를 둘 수 있음

Main Workspace 하단
- 시작하기 버튼 표시
- 사용자가 이 버튼을 눌러야 다음 단계로 감

Inspector
- 자세한 파일 메타정보 표시
- 선택한 설정(FPS, 언어, 템플릿, 모델)에 대한 설명 표시
- 설정 변경 시 관련 설명이 따라오도록 구성 가능

### Ready에서 확정된 UX 방향

Ready 단계는 중앙에 과도한 메타정보를 쌓는 화면이 아니다.

중앙은 다음 역할에 집중한다.

    내가 올린 파일이 맞는지 확인
    지금 설정으로 작업을 시작해도 되는지 확인
    시작 버튼을 누르게 유도

자세한 정보와 설명은 오른쪽 Inspector가 담당한다.

---

## 5. 상단 설정 UI 배치 기준

상단에 설정 UI를 배치하는 것은 현재 DizzyFlow의 방향에서
표준적이고 안전한 선택으로 본다.

이유:
사용자는 파일을 넣기 전부터
이 앱이 작업 규격을 이해하고 있다는 인상을 받을 수 있다.

초기 버전 기준 결정은 다음과 같다.

    상단 설정 바 유지
    Idle / Ready 모두에서 수정 가능
    마지막 사용값 기억
    기존 문서를 클릭한 경우에도 해당 문서에 매핑된 값 표시
    해당 값이 없으면 초기값 사용

디자인적으로는 중앙 드롭 영역을 방해하지 않도록
상단 설정 바가 너무 무겁게 보이지 않게 처리하는 것이 좋다.

예:
- 세미 투명 톤
- Inspector와 연결되는 낮은 대비 톤
- 메인 콘텐츠보다 덜 튀는 시각 우선순위

---

## 6. Document 생성 시점

새 Document 폴더는 파일 업로드 시점에 생성하지 않는다.

반드시 Ready 상태에서 사용자가
"시작하기" 버튼을 눌렀을 때 생성한다.

이 결정의 이유는 다음과 같다.

    파일만 넣어보고 앱을 닫는 경우 불필요한 문서가 쌓이지 않음
    사이드바가 지저분해지는 것을 방지
    사용자가 작업이 공식 등록되는 순간을 명확히 인식 가능

즉 흐름은 다음과 같다.

    Idle
    -> 파일 선택
    -> Ready
    -> 시작하기 클릭
    -> Document 생성
    -> Processing 진입

---

## 7. 기존 문서 선택 시 동작 기준

사이드바에서 기존 문서를 클릭하면
상단 설정 바에는 해당 문서에 매핑된 설정값이 반영된다.

즉 다음과 같은 흐름이다.

    기존 문서 선택
    -> 상단에 그 문서의 FPS / 언어 / FCP 템플릿 / 전사 모델 표시
    -> 값이 없으면 초기값 사용

사용자는 이 값들을 수정할 수 있다.

현재 문서에서는 우선 다음까지만 확정한다.

    기존 문서를 선택해도 상단 설정 UI는 유지된다
    해당 문서에 연결된 값이 선택된 상태로 보인다
    값이 없으면 초기값으로 돌아간다
    사용자는 수정 가능하다

---

## 8. 요약

현재까지 확정된 Idle / Ready 기준은 다음과 같다.

    1. 앱 시작 위치는 Home이다
    2. Sidebar에는 기존 Document 목록이 표시된다
    3. Idle에서는 파일 업로드 전에도 주요 설정 4개를 미리 수정 가능하다
    4. 이 설정값은 마지막 사용값을 기억한다
    5. 파일 업로드 후 Ready 상태가 된다
    6. Ready에서는 중앙에 간단한 파일 확인 정보만 표시한다
    7. 자세한 메타정보와 설정 설명은 Inspector가 담당한다
    8. 시작하기 버튼을 눌러야 실제 작업이 시작된다
    9. Document 폴더는 시작하기를 누른 시점에 생성된다
    10. 기존 문서를 클릭하면 상태(Completed / Failed / Cancelled)에 맞는 화면을 보여준다
