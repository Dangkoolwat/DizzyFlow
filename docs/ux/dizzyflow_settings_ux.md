# DizzyFlow Settings UX Notes
Last updated: 2026-04-04

이 문서는 DizzyFlow의 Settings 화면에 대한 UX/UI 기준을 정리한 문서다.
다른 에이전트가 구현 시 오해하지 않도록
화면 구조, 카테고리 책임, 모델 관리 방식, Safe Lock 규칙, Inspector 가이드를 함께 설명한다.

---

## 1. Settings의 역할

Settings는 단순한 옵션 창이 아니다.

DizzyFlow에서 Settings는 다음 역할을 가진다.

    실행 환경 구성
    전사 관련 동작 설정
    모델 자산 관리
    전처리 옵션 관리
    앱 정보 및 라이선스 확인

즉 Settings는 단순 preference panel이 아니라
DizzyFlow의 실행 환경과 엔진 자산을 관리하는 운영 워크스페이스로 본다.

---

## 2. 진입 방식과 화면 구조

Settings는 별도의 팝업 창이 아니다.

사용자는 Sidebar 하단의 Settings 버튼을 눌러
앱 내부의 Settings 화면으로 진입한다.

기본 구조는 다음과 같다.

    Sidebar | Settings Main | Inspector

즉 전체 앱의 3패널 문법은 유지하되,
Settings Main 내부는 2-column 구조를 사용한다.

    좌측: 설정 카테고리 목록
    우측: 선택한 카테고리의 세부 설정

---

## 3. Settings 메인 구조

Settings Main은 다음과 같은 2-column 구조를 유지한다.

### 좌측 카테고리 목록

초기 버전 기준 카테고리는 다음과 같다.

    1. General
    2. VAD
    3. Preprocessor
    4. Models
    5. About & License

### 우측 세부 설정 영역

좌측에서 선택한 카테고리에 따라
우측 영역의 내용이 바뀐다.

우측 영역은 다음 역할을 가진다.

    현재 카테고리 설명
    실제 설정 UI
    저장 버튼 또는 액션 버튼
    진행 상태 표시

---

## 4. 이번 버전에서 제외하는 항목

AI 편집(GEM4)과 고급 버전 관리는 이번 버전에서 제외한다.

이유:
아직 기능 구현과 구조가 완전히 고정되지 않았기 때문이다.

즉 이번 Settings 문서에는 GEM4 카테고리를 포함하지 않는다.
또한 고급 버전 관리 관련 설정도 다루지 않는다.

이 두 기능은 차기 버전(2.0)에서 별도 논의 및 설계를 진행한다.

---

## 5. Inspector 역할

Settings 화면에서도 Inspector는 유지한다.

Inspector는 read-only 보조 패널로 사용하며,
설정값 자체를 편집하는 공간이 아니다.

역할:

    현재 카테고리에 대한 설명
    Tips! 형식의 자유로운 가이드
    무거운 작업 중 상태 표시
    경고 또는 주의 문구 제공

### 표현 방식

Inspector 설명은 딱딱한 기술 문서형이 아니라
"Tips!" 형식의 짧고 친절한 가이드로 제공한다.

예시:

General
- Tips! 완료된 작업 자동 삭제를 켜면 문서 폴더가 너무 많이 쌓이지 않도록 정리할 수 있습니다.

VAD
- Tips! 무음 구간 기준을 너무 짧게 잡으면 문장이 너무 잘게 끊어질 수 있습니다.

Preprocessor
- Tips! 전처리기를 켜면 더 안정적인 전사 결과를 얻을 수 있지만 처리 시간은 조금 더 늘어날 수 있습니다.

Models
- Tips! 모델마다 지원 언어와 성능이 다릅니다. 현재 Mac에서 지원되지 않는 모델은 비활성화되어 보입니다.

About & License
- Tips! 사용 중인 오픈소스와 앱 버전 정보를 여기서 확인할 수 있습니다.

## 6. General 카테고리

General은 앱의 일반 동작과 문서 관리 정책을 다룬다.

초기 버전 기준 포함 항목은 다음과 같다.

    Remove Symbol
    Model Download Location
    완료 작업 자동 삭제

### 6.1 Remove Symbol

자동 생성된 문서에서 특정 기호를 정리하는 옵션이다.

예시 대상 기호:

    [ ]
    ( )
    기타 지정된 특수문자

초기 버전에서는 복잡한 규칙 편집보다
미리 정의된 정리 옵션 중심으로 시작하는 것이 안전하다.

### 6.2 Model Download Location

모델이 저장되는 기본 위치를 보여주고,
필요 시 변경할 수 있게 한다.

경로를 변경할 때는
기존 모델을 함께 이동할지 확인 단계를 먼저 거친다.

기본 흐름:

    1. 새 경로 선택
    2. 기존 모델도 함께 이동할지 확인
    3. 확인 후 실제 작업 시작

이 작업은 시간이 오래 걸릴 수 있으므로
Settings Safe Lock 대상이 된다.

### 6.3 완료 작업 자동 삭제

완료된 작업 문서를 자동으로 정리하는 옵션이다.

선택지는 다음과 같다.

    7일
    15일
    30일
    삭제 안함

이 기능은 문서 폴더가 과도하게 쌓이는 것을 방지하기 위한
운영 편의 기능으로 본다.

### 6.4 저장 정책

General 카테고리는 실제 동작에 영향을 미치는 설정이므로
저장 버튼을 둔다.

예시:

    [저장]

## 7. VAD 카테고리

VAD는 음성 활동 감지 설정을 다룬다.

이 카테고리는 단순 사용자 옵션이라기보다
전사 품질과 세그먼트 분할 품질에 직접 영향을 주는 설정 영역이다.

초기 버전 기준 항목은 다음과 같다.

    Threshold
    Min Speech Duration
    Min Silence Duration
    Max Speech Duration

각 항목은 드롭다운 메뉴로 제공한다.

기본값은 선택지나 레이블에서
괄호 형태로 명시한다.

예시:

    Threshold (기본값)
    Min Silence Duration (기본값)

### 저장 정책

VAD는 실제 처리 결과에 영향을 주는 설정이므로
저장 버튼을 둔다.

예시:

    [저장]

---

## 8. Preprocessor 카테고리

Preprocessor는 ffmpeg 기반 전처리 설정을 다루는 영역이다.

"ffmpeg"라는 내부 기술 이름을 전면에 내세우기보다
영상 편집자에게 더 익숙한 "Preprocessor"라는 명칭을 사용한다.

### 8.1 Preprocessor의 목적

Preprocessor는 더 안정적인 전사 결과를 위해
오디오를 가공하는 단계다.

예시 안내 문구:

    전처리기는 더 안정적인 전사 결과를 위해 오디오를 가공합니다.
    입력 파일은 엔진에 맞는 형식으로 자동 최적화될 수 있습니다.

### 8.2 전체 사용 여부

Preprocessor는 외부 프로그램(ffmpeg)을 실행하기 때문에
추가 시간이 발생할 수 있다.

따라서 사용자는 전처리 단계의 사용 여부를 명시적으로 제어할 수 있어야 한다.

초기 버전에는 다음 토글을 둔다.

    [ ] Preprocessor 사용

설명:

    활성화하면 ffmpeg 기반 전처리 단계를 수행
    비활성화하면 추가 품질 개선 전처리를 생략
    처리 시간은 늘어날 수 있지만 전사 품질이 더 안정적일 수 있음

### 8.3 세부 옵션

다음 옵션은 Preprocessor 사용이 켜진 경우에만 활성화된다.

    Audio Normalization
    Noise Reduction
    Channel Mixdown

이 옵션들은 사용자가 이해 가능한 품질 개선 기능만 노출한다.

### 8.4 내부 자동 처리와의 구분

Preprocessor를 끄더라도
엔진 입력 규격에 맞추기 위한 최소 변환은 내부적으로 수행될 수 있다.

즉 의미는 다음과 같다.

    Preprocessor OFF
    -> 추가 품질 개선 전처리 생략
    -> 필요한 최소 규격 정규화는 내부 자동 처리 가능

    Preprocessor ON
    -> ffmpeg 기반 품질 개선 전처리 수행
    -> 선택한 옵션 반영

### 8.5 저장 정책

Preprocessor는 실제 처리 흐름에 영향을 주는 설정이므로
저장 버튼을 둔다.

예시:

    [저장]

## 9. Models 카테고리

Models는 가장 중요한 Settings 섹션 중 하나다.

이 카테고리는 단순 설정이 아니라
모델 자산을 관리하는 운영 화면 역할을 한다.

모델 정보는 JSON 기반 데이터에서 읽어와
동적으로 구성하는 것을 기본 전제로 한다.

즉 모델 목록을 하드코딩하기보다
registry 또는 metadata 기반으로 관리하는 방향을 권장한다.

### 9.1 화면 구조

Models 우측 세부 영역에서는
큰 카테고리 구분을 먼저 보여주고,
각 영역 안에 세부 모델 카드를 배치한다.

각 모델 카드에는 최소 다음 정보를 포함한다.

    이름
    주요 기능
    크기
    지원 언어
    지원 플랫폼
    현재 상태
    다운로드 버튼 또는 삭제 버튼

### 9.2 카드 액션 규칙

모델이 설치되어 있지 않으면

    다운로드 버튼 표시

모델이 이미 설치되어 있으면

    삭제 버튼 표시

버튼은 카드 단위 액션으로 처리한다.

### 9.3 다운로드 진행 표시

모델 다운로드는 전역 팝업보다
해당 모델 카드 아래에서 직접 진행 상태를 보여주는 방식을 우선한다.

예시:

    다운로드 중...
    진행 바 표시
    퍼센트 또는 단계 표시

이 방식의 장점:

    사용자는 어떤 모델이 현재 작업 중인지 바로 이해 가능
    전체 Settings 화면을 과하게 모달화하지 않아도 됨
    모델 관리 흐름이 카드 중심으로 자연스럽게 보임

### 9.4 플랫폼 호환성 규칙

모델 카드는 현재 Mac 환경과의 호환성을 반영해야 한다.

예시:

    Sherpa-onnx
    -> Intel / Apple Silicon 모두 가능

    WhisperKit
    -> Intel Mac 사용 불가

### 9.5 WhisperKit 처리 규칙

WhisperKit은 Intel Mac에서 사용할 수 없다.

따라서 Intel Mac 사용자에게는
WhisperKit 카드 전체를 Dim 처리하고 비활성화한다.

권장 표시 방식:

    카드 전체 Dim 처리
    클릭 불가
    다운로드 버튼 비활성화
    삭제 버튼 비활성화
    지원 불가 안내 문구 표시

예시 문구:

    Apple Silicon required
    이 모델은 Intel Mac에서 사용할 수 없습니다.

즉 단순히 버튼만 막는 것이 아니라,
왜 사용할 수 없는지 사용자에게 즉시 이해되게 해야 한다.

### 9.6 저장 정책

Models는 카드 단위 액션 중심 화면이므로
별도의 전체 저장 버튼을 두지 않는다.

다운로드 / 삭제 / 상태 변화는 각 카드에서 직접 처리한다.

## 10. About & License 카테고리

이 카테고리는 읽기 전용 정보 화면이다.

초기 버전 기준 포함 항목은 다음과 같다.

    앱 이름
    버전 정보
    연락처 또는 저장소 정보
    사용된 오픈소스 및 라이선스 목록

라이선스 목록은 우측 세부 영역 안에서
스크롤 가능한 형태로 보여줄 수 있다.

### 저장 정책

읽기 전용 화면이므로 저장 버튼은 없다.

## 11. Settings Safe Lock

Settings에서도 무거운 작업이나 비가역적 작업이 발생할 수 있다.

예시:

    모델 저장 위치 변경
    대량 파일 이동
    모델 삭제
    기타 시간이 오래 걸리는 운영 작업

이 경우 Settings 전용 Safe Lock을 적용할 수 있다.

### 목적

    작업 중 상태 꼬임 방지
    설정값 변경 충돌 방지
    사용자가 앱이 실제로 작업 중임을 인지하게 함

### 잠그는 대상

Safe Lock 중에는 다음 인터랙션을 일시적으로 막을 수 있다.

    Settings 카테고리 전환
    현재 세부 설정 수정
    Sidebar의 다른 메뉴(Home, Documents) 이동

작업 성격에 따라
전역 잠금 또는 해당 설정 영역 잠금을 선택할 수 있다.

### 진행 표시

작업이 크고 시간이 오래 걸리는 경우에는
중앙 팝업 또는 상태 레이어를 통해 다음을 보여줄 수 있다.

    현재 작업명
    진행 바
    단계 표시
    성공 / 실패 메시지

예시:

    파일 이동 중... (4/10)

### 취소 버튼

작업 종류에 따라 취소 버튼을 둘 수 있다.

단,
파일 이동처럼 중간 취소가 위험한 작업은
취소보다 진행 상태 표시와 결과 확인에 집중하는 것이 더 안전할 수 있다.

---

## 12. 작업 종료 피드백

Settings에서 수행한 무거운 작업은
끝났을 때 확실한 피드백을 제공해야 한다.

### 성공 시

예시:

    모든 모델이 성공적으로 이동되었습니다.

    [확인]

확인을 누르면
상태 레이어 또는 팝업이 닫히고
Settings 화면이 갱신된다.

### 실패 시

예시:

    용량 부족으로 이동에 실패했습니다.

실패 시에는
사용자가 다음 행동을 판단할 수 있도록
구체적인 이유를 남기는 것을 우선한다.

## 13. 카테고리별 Inspector Tips 예시

### General
- Tips! 완료된 작업 자동 삭제를 켜면 문서 폴더가 너무 많이 쌓이지 않도록 정리할 수 있습니다.
- Tips! 모델 저장 위치를 변경하면 디스크 사용량과 이동 시간이 영향을 받을 수 있습니다.

### VAD

- Tips! 기본값에서 크게 벗어나기 전에는 현재 결과를 먼저 비교해 보는 것이 좋습니다.

### Preprocessor

- Tips! 전처리기를 끄더라도 엔진 입력에 필요한 최소 변환은 내부적으로 처리될 수 있습니다.

- Tips! 현재 Mac에서 사용할 수 없는 모델은 비활성화되어 보입니다.

## 15. 요약

현재까지 확정된 Settings 방향은 다음과 같다.

    1. Settings는 Sidebar 하단에서 진입하는 앱 내부 화면이다
    2. 전체 앱의 3패널 구조를 유지한다
    3. Settings Main은 좌측 카테고리 / 우측 세부 설정의 2-column 구조다
    4. 이번 버전에서는 AI 편집(GEM4)과 고급 버전 관리를 제외한다
    5. Inspector는 Tips! 형식의 자유로운 가이드를 제공한다
    6. General에는 완료 작업 자동 삭제 기능이 포함된다
    7. VAD는 드롭다운 기반 세부 음성 감지 설정을 제공한다
    8. Preprocessor는 ffmpeg 기반 전처리 설정이며 전체 사용 여부 토글이 있다
    9. Models는 카드 기반 모델 자산 관리 화면이다
    10. WhisperKit은 Intel Mac에서 전체 Dim 처리 및 비활성화된다
    11. Settings의 무거운 작업에는 Safe Lock 규칙을 적용할 수 있다
    12. 저장 버튼은 General / VAD / Preprocessor에만 둔다
